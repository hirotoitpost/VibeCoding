# 🔧 Git 規約スキル（VibeCoding プロジェクト）

**このスキルは Git コミット・PR メッセージの規約に特化**しています。  
📋 **ドキュメント管理戦略は別スキルを参照してください** → [ドキュメント管理スキル](.instructions-doc.md)

## 📋 対象ドキュメント

### 1. **コミットメッセージ規約**

- **ファイル**: [GIT_COMMIT_CONVENTION.md](GIT_COMMIT_CONVENTION.md)
- **用途**: コミットメッセージの形式・Type 定義・実例を確認
- **対象**: すべての feature / fix / docs ブランチ
- **参考**: Type（feat, fix, docs, refactor, test, chore, restore）の定義

### 2. **プルリクエスト規約**

- **ファイル**: [PULL_REQUEST_GUIDELINES.md](PULL_REQUEST_GUIDELINES.md)
- **用途**: PR メッセージのテンプレート・チェックリスト・タイプ別ガイド
- **対象**: PR 作成時のタイトル、説明文、チェック項目
- **参考**: よくある間違いと修正セクション

## 🔄 使用シーン

### シーン 1: コミットメッセージ作成時

```bash
git commit -m "feat(ID 027): AviUtl Exo ファイル INI 形式書き直し"
```

→ [GIT_COMMIT_CONVENTION.md](GIT_COMMIT_CONVENTION.md) の Type・Scope・Subject セクションを参照

### シーン 2: PR 作成時

```
【PR メッセージテンプレート】
## 変更の概要
[PR で実装した内容の要約]

## 変更の詳細
[何を変更したのか、なぜ必要か]

## セッション・Work ID
- Session: XX
- ID: XXX
```

→ [PULL_REQUEST_GUIDELINES.md](PULL_REQUEST_GUIDELINES.md) のテンプレートセクションを参照

### シーン 3: レビューコメント対応時

→ [PULL_REQUEST_GUIDELINES.md](PULL_REQUEST_GUIDELINES.md) の「PR レビュー後」項目を参照

## 📚 クイックリファレンス

### コミット Type 一覧（簡略版）

| Type         | 用途             | 例                                          |
| ------------ | ---------------- | ------------------------------------------- |
| **feat**     | 新機能           | `feat(ID 027): Exo INI 形式書き直し`        |
| **fix**      | バグ修正         | `fix(generate_exo): テキスト形式判定エラー` |
| **docs**     | ドキュメント     | `docs(Session 27): ドキュメント整理完了`    |
| **refactor** | コード改善       | `refactor(app): 関数抽出`                   |
| **test**     | テスト追加       | `test(api): エンドポイント検証`             |
| **chore**    | ビルド・依存関係 | `chore(deps): npm update`                   |
| **restore**  | ファイル復活     | `restore: Session 2-4 記録復活`             |

詳細は [GIT_COMMIT_CONVENTION.md](GIT_COMMIT_CONVENTION.md#-type必須) を参照

### PR メッセージの構成（簡略版）

1. **変更の概要** - 60-80 文字で簡潔に
2. **変更の詳細** - 何を・なぜ変更したのか
3. **セッション・Work ID** - Session XX, ID XXX
4. **テスト実施** - チェックボックス（ローカルテスト、ドキュメント確認）
5. **チェックリスト** - 規約遵守、ファイル除外等

詳細は [PULL_REQUEST_GUIDELINES.md](PULL_REQUEST_GUIDELINES.md#-pr-メッセージテンプレート) を参照

## ✅ 重要なチェック項目

**コミット時**:

- [ ] Type が正しいか（feat/fix/docs/refactor/test/chore/restore）
- [ ] Scope（ID・ファイル名）が含まれているか
- [ ] Subject は命令形で記述されているか

**PR 作成時**:

- [ ] タイトルが明確か（例: `feat(ID 027): Exo INI 形式書き直し`）
- [ ] 説明文（body）が充実しているか（What/Why/How）
- [ ] ファイル差分が最小限か（無関連なファイルが含まれていないか）
- [ ] .env や出力ファイルが除外されているか
