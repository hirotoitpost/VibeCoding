# アプリケーション候補集 - VibeCoding 学習プロジェクト

## 概要
このドキュメントはフェーズ3（実装フェーズ）で取り組むプロジェクト候補を、  
難易度・学習効果・実施時間の観点から厳選しました。

---

## 難易度の定義

| 難易度 | 説明 | 環境構築 | 複雑性 | 外部依存 * |
|-------|------|-------|-------|---------|
| ⭐ 基礎 | 単言語、標準ライブラリ | 簡単 | 低 | 少ない |
| ⭐⭐ 初級 | 複数ライブラリ、シンプルな統合 | 中程度 | 中 | 中程度 |
| ⭐⭐⭐ 中級 | 複数サービス、インフラ要素 | 複雑 | 高 | 多い |
| ⭐⭐⭐⭐ 発展 | 複数プロジェクト統合、本番品質 | 高度 | 非常に高 | 非常に多い |

**外部依存***: API、ハードウェア、インフラ等への依存度

### 候補 1A: 天気情報取得ツール（推奨）

**説明**:  
OpenWeatherMap の API を使用して、指定都市の天気情報を取得・表示するコンソールツール。

**技術スタック**:
- **言語**: Python
- **ライブラリ**: requests（HTTP クライアント）
- **外部API**: OpenWeatherMap
- **ローカルストレージ**: なし

**要件**:
```
入力: 都市名（日本語）
出力: 気温、天気状態、降水確率（テキスト出力）

追加機能:
- エラーハンドリング（API エラー、ネットワーク）
- キャッシング（5分間）
- 複数都市の一括取得
```

**指示文の例**:
```
「OpenWeatherMap のAPIを使用して、
入力された都市名の現在の天気情報（気温、状態、降水確率）を
日本語で画面に表示する Python スクリプトをお願いします。

要件:
- エラーハンドリングを含むこと（API失敗時、ネットワーク問題）
- 5分間のキャッシング機能
- 複数都市を連続入力可能

API キーは環境変数 OPENWEATHER_API_KEY から読み込んでください」
```

**Vibe Coding の学習ポイント**:
- ✅ API 統合の基本
- ✅ エラーハンドリング
- ✅ 環境変数の使用
- ✅ 外部ライブラリの依存管理

**AI の得意分野**:
- ✅ コード全体の実装
- ✅ エラーハンドリングの追加

**AI の不得意分野**:
- ❌ OpenWeatherMap アカウント作成
- ❌ API キー取得
- ⚠️ キャッシング戦略の詳細設計

**推定期間**: 2-4 時間  
**初回体験に最適**: ⭐⭐⭐⭐⭐

## 🎯 ID 017 プロジェクト名：「AviUtl + PSDToolKit + VOICEVOX 動画自動生成システム」

### 概要
**春日部つむぎ解説動画システム（ID 016）の移行版**。  
Shoost は PSD ファイル非対応であることが判明したため、AviUtl + PSDToolKit + VOICEVOX を用いた
**完全自動動画生成パイプライン**へ移行する。

**フェーズ**: 3.3 拡張 / フェーズ 4 準備  
**難易度**: ⭐⭐⭐ (中級)  
**ステータス**: 🔄 実装中（Session 19~）

### 技術スタック
| コンポーネント | 役割 | 技術 |
|-------------|------|------|
| **AviUtl** | 動画編集・エンコード | スクリプト API・拡張プラグイン |
| **PSDToolKit** | PSD → 画像変換・立ち絵管理 | AviUtl プラグイン |
| **VOICEVOX** | 音声合成エンジン | 春日部つむぎ (Speaker ID: 14) REST API |
| **PowerShell** | 統合制御・自動化 | パイプライン全体の制御スクリプト |

### 動画生成パイプライン

```
[入力]
  台本テキスト (.txt / .md)
       ↓
[1. 音声生成]
  VOICEVOX API (localhost:50021)
  → 春日部つむぎの音声 (.wav)
       ↓
[2. AviUtl + PSDToolKit]
  PSD 立ち絵 → PSDToolKit で画像化
  音声 + 立ち絵 → AviUtl でタイムライン制御
  字幕テキスト → AviUtl でオーバーレイ
       ↓
[3. エンコード出力]
  MP4 / WebM (YouTube 投稿・ナレッジシェア用)
```

### 要件

```
Phase 1: 環境セットアップ
  - AviUtl 本体インストール確認 (AVIUTL_ROOT 環境変数)
  - PSDToolKit プラグイン導入手順
  - VOICEVOX 起動・API 疎通確認スクリプト

Phase 2: VOICEVOX 音声生成
  - 台本テキスト → 音声ファイル生成スクリプト (PowerShell)
  - 複数行台本の一括生成
  - 音声ファイル命名規則 (01_intro.wav, 02_body.wav, ...)

Phase 3: AviUtl プロジェクト自動生成
  - Exo ファイル (AviUtl プロジェクト) テンプレート生成スクリプト
  - 音声タイムライン自動配置
  - 字幕テキストオーバーレイ自動生成

Phase 4: 動画エンコード・出力
  - AviUtl CLI 制御スクリプト
  - 最終 MP4 出力・品質設定

最終成果物:
  - generate_voice.ps1 (台本 → 音声一括生成)
  - generate_exo.ps1 (Exo プロジェクトファイル自動生成)
  - encode_video.ps1 (エンコード実行)
  - run_all.ps1 (ワンコマンドで全工程実行)
  - VibeCoding プロジェクト解説スクリプト集 (台本テンプレート)
```

### Vibe Coding の学習ポイント
- ✅ **Windows デスクトップアプリケーションの CLI / API 制御**
- ✅ **VOICEVOX REST API の PowerShell Python 統合**
- ✅ **AviUtl Exo ファイル形式の自動生成**
- ✅ **パイプライン設計（テキスト → 音声 → 動画）**
- ✅ **環境変数（AVIUTL_ROOT）活用**
- ✅ **VibeCoding 全プロジェクトのナレッジシェア化**

### AI の得意分野
- ✅ VOICEVOX API 呼び出しスクリプト
- ✅ Exo ファイルフォーマット生成スクリプト
- ✅ PowerShell パイプライン設計
- ✅ 台本テンプレート・解説スクリプト執筆
- ✅ 環境変数読み込み・検証スクリプト

### AI が対応困難な分野
- ❌ AviUtl GUI 上での手動セットアップ
- ❌ PSDToolKit の初期設定（GUI）
- ⚠️ エンコード品質の細かいチューニング
