# アプリケーション候補集 - VideCoding 学習プロジェクト

## 概要
このドキュメントはフェーズ3（実装フェーズ）で取り組むプロジェクト候補を、  
難易度・学習効果・実施時間の観点から厳選しました。

---

## 難易度の定義

| 難易度 | 説明 | 環境構築 | 複雑性 | 外部依存 * |
|-------|------|-------|-------|---------|
| ⭐ 基礎 | 単言語、標準ライブラリ | 簡単 | 低 | 少ない |
| ⭐⭐ 初級 | 複数ライブラリ、シンプルな統合 | 中程度 | 中 | 中程度 |
| ⭐⭐⭐ 中級 | 複数サービス、インフラ要素 | 複雑 | 高 | 多い |
| ⭐⭐⭐⭐ 発展 | 複数プロジェクト統合、本番品質 | 高度 | 非常に高 | 非常に多い |

**外部依存***: API、ハードウェア、インフラ等への依存度

## 🌟 フェーズ3.1: 基礎プロジェクト候補（難易度 ⭐）

### 候補 1A: 天気情報取得ツール（推奨）

**説明**:  
OpenWeatherMap の API を使用して、指定都市の天気情報を取得・表示するコンソールツール。

**技術スタック**:
- **言語**: Python
- **ライブラリ**: requests（HTTP クライアント）
- **外部API**: OpenWeatherMap
- **ローカルストレージ**: なし

**要件**:
```
入力: 都市名（日本語）
出力: 気温、天気状態、降水確率（テキスト出力）

追加機能:
- エラーハンドリング（API エラー、ネットワーク）
- キャッシング（5分間）
- 複数都市の一括取得
```

**指示文の例**:
```
「OpenWeatherMap のAPIを使用して、
入力された都市名の現在の天気情報（気温、状態、降水確率）を
日本語で画面に表示する Python スクリプトをお願いします。

要件:
- エラーハンドリングを含むこと（API失敗時、ネットワーク問題）
- 5分間のキャッシング機能
- 複数都市を連続入力可能

API キーは環境変数 OPENWEATHER_API_KEY から読み込んでください」
```

**Vibe Coding の学習ポイント**:
- ✅ API 統合の基本
- ✅ エラーハンドリング
- ✅ 環境変数の使用
- ✅ 外部ライブラリの依存管理

**AI の得意分野**:
- ✅ コード全体の実装
- ✅ エラーハンドリングの追加

**AI の不得意分野**:
- ❌ OpenWeatherMap アカウント作成
- ❌ API キー取得
- ⚠️ キャッシング戦略の詳細設計

**推定期間**: 2-4 時間  
**初回体験に最適**: ⭐⭐⭐⭐⭐

## ⭐⭐ フェーズ3.2: 初級プロジェクト候補（難易度 ⭐⭐）

### 候補 2A: シンプルな Web 家計簿アプリ（推奨）

**説明**:  
収支を記録・管理する Web アプリ。  
フロントエンド + バックエンド + データベースの統合体験。

**技術スタック**:
- **フロントエンド**: React, React Router, Axios
- **バックエンド**: Node.js + Express
- **データベース**: SQLite
- **ローカル実行**: npm + node

**要件**:
```
機能:
- 収支記録：日付、カテゴリ、金額、説明の入力
- 表示：月別集計、カテゴリ別集計、グラフ表示
- 編集・削除機能
- CSV エクスポート

セキュリティ:
- 入力値のバリデーション
- SQL インジェクション対策
```

**指示文の例**:
```
「React + Node.js/Express で家計簿 Web アプリを作成してください。

要件:
【フロントエンド】
- React で UI を構築
- 収支入力フォーム、月別集計表示
- グラフ表示（Chart.js 推奨）

【バックエンド】
- REST API（Express)
- SQLite データベース
- バリデーション・エラーハンドリング

【データベース】
- テーブル: users, transactions
- ユーザー認証は簡易版（未実装でOK）

【デプロイ】
- docker-compose.yml で ワンコマンド起動可能に
」
```

**Vibe Coding の学習ポイント**:
- ✅複数サービスの統合
- ✅ API 設計の経験
- ✅ データベース基礎
- ✅ フロントエンド・バックエンド分離

**AI の得意分野**:
- ✅ 各サービスのコード生成
- ✅ API エンドポイント設計

**AI が対応困難な分野**:
- ❌ Node.js、npm のインストール手順
- ❌ パッケージのバージョン互換性問題
- ⚠️ データベーススキーマの最適化

**推定期間**: 3-5 日  
**多言語・複数フレームワーク体験**: ⭐⭐⭐⭐⭐
