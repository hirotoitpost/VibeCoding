## 🌟 フェーズ 3.1: 基礎プロジェクト（難易度 ⭐）

### 候補 1A: 天気情報取得ツール（推奨）

**説明**: OpenWeatherMap API を使用して、指定都市の天気情報を取得・表示するコンソールツール。

**技術スタック**:
- **言語**: Python
- **ライブラリ**: requests（HTTP クライアント）
- **外部API**: OpenWeatherMap
- **ローカルストレージ**: なし

**要件**:
- 入力: 都市名（日本語） / 出力: 気温、天気状態、降水確率
- エラーハンドリング（API エラー、ネットワーク）
- キャッシング（5分間） / 複数都市の一括取得

**指示文の例**:
```
「OpenWeatherMap の API を使用して、  
入力された都市名の現在の天気情報（気温、状態、降水確率）を  
日本語で画面に表示する Python スクリプトをお願いします。

要件:
- エラーハンドリングを含むこと（API 失敗時、ネットワーク問題）
- 5 分間のキャッシング機能
- 複数都市を連続入力可能

API キーは環境変数 OPENWEATHER_API_KEY から読み込んでください」
```

**Vibe Coding の学習ポイント**:
- ✅ API 統合の基本
- ✅ エラーハンドリング
- ✅ 環境変数の使用
- ✅ 外部ライブラリの依存管理

**推定期間**: 2-4 時間  
**初回体験に最適**: ⭐⭐⭐⭐⭐

## ⭐⭐ フェーズ 3.2: 初級プロジェクト（難易度 ⭐⭐）

### 候補 2A: シンプルな Web 家計簿アプリ（推奨）

**説明**: 収支を記録・管理する Web アプリ。フロントエンド + バックエンド + データベースの統合体験。

**技術スタック**:
- **フロントエンド**: React, React Router, Axios
- **バックエンド**: Node.js + Express
- **データベース**: SQLite
- **ローカル実行**: npm + node

**要件**:
- 収支記録：日付、カテゴリ、金額、説明の入力
- 表示：月別集計、カテゴリ別集計、グラフ表示
- 編集・削除機能 / CSV エクスポート
- 入力値バリデーション / SQL インジェクション対策

**指示文の例**:
```
「React + Node.js/Express で家計簿 Web アプリを作成してください。

要件:
【フロントエンド】
- React で UI を構築
- 収支入力フォーム、月別集計表示
- グラフ表示（Chart.js 推奨）

【バックエンド】
- REST API（Express)
- SQLite データベース
- バリデーション・エラーハンドリング

【データベース】
- テーブル: users, transactions
- ユーザー認証は簡易版（未実装でOK）

【デプロイ】
- docker-compose.yml でワンコマンド起動可能に」
```

**Vibe Coding の学習ポイント**:
- ✅ 複数サービスの統合
- ✅ API 設計の経験
- ✅ データベース基礎
- ✅ フロントエンド・バックエンド分離

**推定期間**: 3-5 日
