## 1. GitHub Copilot

### 1.1 概要
- **形態**: VS Code 拡張
- **動作**: コード補完（自動提案）
- **特徴**: 軽量、即座、コンテキスト理解優秀
- **料金**: 無料版あり（機能限定）、有料版

### 1.2 セットアップ

#### インストール
```
VS Code マーケットプレイス → 「GitHub Copilot」検索 → インストール
```

#### ログイン
```
1. GitHub アカウントでサインイン
2. GitHub Copilot ライセンス確認
3. VS Code で認証完親
```

### 1.3 使用パターン

#### パターン A: コード補完
```python
# ユーザーが入力
def get_weather_data(city):
    # Copilot が以下を提案
    import requests
    url = f"https://api.openweathermap.org/data/2.5/weather?q={city}"
    response = requests.get(url)
    return response.json()
```

**指示方法**: 
- コメントで意図を記載
- 関数名から推測させる
- 既存コードから流れを推測させる

**操作**:
```
Tab キーで提案を受け入れ
Esc キーで提案をキャンセル
Ctrl+Enter で他の提案案を表示
```

#### パターン B: 複数行補完
```python
# データクラスの初期化
def validate_input(data):
    # Tab × 複数回で段階的に補完
    if not data:
        return False
    if len(data) > 100:
        return False
    return True
```

### 1.4 工夫・コツ

| コツ | 説明 | 例 |
|------|------|-----|
| **コメント活用** | 意図を明確に | `# エラーハンドリングを追加` |
| **型ヒント** | 型情報を提供 | `def process(data: dict) -> str:` |
| **ライブラリ import** | インポート後が高精度 | `import requests` の後が高精度 |
| **既存パターン利用** | 前のコードから学習 | 同じパターンの関数を先に書く |
| **短めの関数** | 小さい単位で指示 | 100行超は分割を検討 |

### 1.5 期待される出力品質

| シーン | 品質 | 注意点 |
|-------|------|-------|
| **ボイラープレート** | 高 ✅ | そのまま使用可 |
| **CRUD操作** | 高 ✅ | テスト推奨 |
| **複雑なロジック** | 中 ⚠️ | レビュー・修正必須 |
| **新しい API** | 低 ❌ | 手作業推奨 |

### 1.6 適用シーン ✅

```
✅ 既存プロジェクトへの機能追加
✅ テストコード作成
✅ ドキュメンテーション補完
✅ 単純な関数実装
```

### 1.7 不向きなシーン ❌

```
❌ 全体的な構造設計
❌ 複雑なアルゴリズム
❌ 複数ファイルの同時指示
```

## 2. Cursor

### 2.1 概要
- **形態**: IDE（VS Code ベース）
- **動作**: チャット型・マルチファイル編集
- **特徴**: 直感的、複数ファイル対応、デバッグ機能
- **料金**: 無料版＋有料プラン

### 2.2 セットアップ

#### インストール
```
1. https://www.cursor.com/ からダウンロード
2. インストール
3. VS Code の設定をインポート可能
```

#### 初期設定
```
1. API キー設定（OpenAI or Claude）
2. 言語モデル選択
3. ショートカット設定
```

### 2.3 使用パターン

#### パターン A: チャット型指示
```
【Cursor チャット】
ユーザー: 「Python で OpenWeatherMap API から天気情報を取得するスクリプトを作成してください。
         エラーハンドリングと 5 分間のキャッシング機能を付けてください。」

Cursor: 
  1. weather_fetcher.py を新規作成
  2. API キー環境変数設定
  3. キャッシング機能（内蔵）
  4. テストコード生成
```

**指示方法**: 自然言語で「何をしたいか」を明確に

#### パターン B: マルチファイル編集
```
【複数ファイルの指示】
ユーザー: 「main.py と utils.py を修正して、
         以下の機能を追加してください：
         - ロギング機能
         - 設定ファイル読み込み」

Cursor: 両ファイルを自動分析＆修正提案
```

#### パターン C: コード修正・リファクタリング
```
【既存コードの改善】
ユーザー: 「このコードを PEP 8 に準拠させてください」

Cursor: 自動リファクタリング提案
```

### 2.4 工夫・コツ

| コツ | 説明 |
|------|------|
| **具体性** | 「Webアプリを作る」→「Flask で CRUD アプリ」 |
| **制約明示** | 「Python 3.9 以上」「テストは pytest」 |
| **段階的指示** | 一度に全部ではなく「まず API」「次にUI」 |
| **修正指示** | 「これが違う」より「こう変えて」が効果的 |
| **ファイル参照** | 「main.py を確認して」で文脈理解 |

### 2.5 期待される出力品質

| シーン | 品質 | 注意点 |
|-------|------|-------|
| **マルチファイル生成** | 高 ✅ | 全体構造の理解が優秀 |
| **リファクタリング** | 高 ✅ | 統一性を保持 |
| **CRUD × DB** | 中 ⚠️ | スキーマ確認推奨 |
| **フロントエンド** | 中 ⚠️ | デザインはテーマ確認 |

### 2.6 適用シーン ✅

```
✅ 新規プロジェクト全体構築
✅ マイクロサービス生成
✅ リファクタリング
✅ 複数ファイルの同時修正
✅ テストコード生成
```

### 2.7 不向きなシーン ❌

```
❌ 細かい記述ニーズ（Copilot の方が向く）
❌ コード補完だけの用途（Copilot で十分）
```

## 3. Claude Code（このチャット）

### 3.1 概要
- **形態**: チャット UI + API
- **動作**: 推論能力高い、複雑な会話
- **特徴**: 複雑な要件理解、段階的実装指示
- **料金**: を通じた使用

### 3.2 使用パターン

#### パターン A: 要件定義から実装まで
```
【チャット型対話】
ユーザー: 「天気情報取得ツールを作りたいです。
         OpenWeatherMap API を使います。
         どうやって進めたらいいですか？」

Claude Code:
  1. 要件ブレークダウン
  2. 技術スタック提案
  3. 段階的実装計画
  4. 実装（段階ごと）
  5. テスト・検証
```

#### パターン B: 複雑な設計相談
```
ユーザー: 「IoT センサーデータを Web ダッシュボードで表示したいです。
         アーキテクチャはどうすべき？」

Claude Code:
  → マイクロサービス設計提案
  → 各層の技術選択理由説明
  → コード生成
```

#### パターン C: エラーデバッグ・修正指示
```
ユーザー: 「このエラーが出ました。[エラーメッセージ]」

Claude Code:
  → 原因分析
  → 修正コード提案
  → 今後の対策提示
```

### 3.3 工夫・コツ

| コツ | 説明 |
|------|------|
| **詳細な背景説明** | 制約・条件をすべて提示 |
| **段階的質問** | 「全体設計」→「個別機能」の順 |
| **コード貼り付け** | エラー、既存コード共有で高精度 |
| **明確な終了条件** | 「テスト成功まで」など、ゴール明示 |
| **フィードバック** | 「これが違う」と修正指示が重要 |

### 3.4 期待される出力品質

| シーン | 品質 | 注意点 |
|-------|------|-------|
| **要件分析** | 非常に高い ⭐⭐⭐ | 完全に頼れる |
| **設計指導** | 非常に高い ⭐⭐⭐ | アーキテクチャ理解優秀 |
| **複雑な実装** | 高 ✅ | 段階的指示で成功 |
| **エラー解析** | 高 ✅ | スタックトレース活用有効 |
| **全体統合** | 高 ✅ | 複数ファイル・複数ツール連携 |

### 3.5 適用シーン ✅

```
✅ 新規プロジェクト全体の計画・指導
✅ 複雑なアーキテクチャ設計
✅ エラーデバッグ・根本原因分析
✅ 段階的な複雑な実装
✅ ナレッジシェア資料作成（このドキュメント）
✅ 学習・教育ガイド
```

### 3.6 不向きなシーン ❌

```
❌ リアルタイムコード補完（Copilot の方が向く）
```

## 5. 効果的な指示文テンプレート

### 5.1 GitHub Copilot 向け

```python
# 【テンプレート】
# [目的: 何をしたいのか]
# [入出力: 型・形式]
# [注意事項: 特殊条件]

# 例：
def fetch_and_cache_data(url: str) -> dict:
    # 目的: URL から JSON データを取得し、1時間キャッシング
    # 入出力: str -> dict
    # 注意: キャッシュキーは URL をハッシュ化
```

### 5.2 Cursor 向け

```
「Python で以下の機能を持つスクリプトを作成してください：

【要件】
1. OpenWeatherMap API から天気情報を取得
2. データベースに保存（SQLite）
3. Web UI で表示（Flask）

【制約】
- Python 3.9 以上
- 外部依存を最小化
- エラーハンドリング必須

【テスト】
- テストケースを pytest で作成
- エラーケースも含める

まず、ファイル構成を提案してください」
```

### 5.3 Claude Code 向け

```
「以下の要件を持つシステムを設計・実装したいです。

【背景】
- IoT センサーから温度・湿度を収集
- リアルタイムでダッシュボード表示
- 異常時にアラート通知

【要件】
1. アーキテクチャ設計（マイクロサービス推奨）
2. 技術選定の根拠
3. 段階的実装計画
4. 実装（段階1: センサー収集）

【質問】
まず、全体設計をアドバイスしてもらえますか？」
```

## 6. 実践フロー（複合利用）

### 6.1 プロジェクト全体フロー

```
Step 1: Claude Code で要件分析・設計
  ↓ デザインドキュメント作成
  
Step 2: Cursor でプロジェクト構造生成
  ↓ main.py, utils.py, config.py 等を自動生成
  
Step 3: GitHub Copilot で細部実装
  ↓ コード補完で効率化
  
Step 4: Claude Code でレビュー・最適化提案
  ↓ リファクタリング指示
  
Step 5: Cursor で一括修正
  ↓ 複数ファイル同時対応
  
Step 6: テスト・デプロイ
```

### 6.2 各段階でのコミット

```
【Commit 1】基本設計・ドキュメント
  Claude Code のアドバイスをまとめたドキュメント

【Commit 2】プロジェクト構造
  Cursor で生成した初期ファイル群

【Commit 3】機能実装
  GitHub Copilot + Claude Code でコード完成

【Commit 4】テスト・最適化
  Cursor でテストコード生成
```

## 7. チェックリスト

### 7.1 指示前の確認

- [ ] 要件が明確に定義できているか

### 7.2 出力後の検証

- [ ] 生成コードが理解できるか
- [ ] テストが通るか

- [ ] スタイルガイド（PEP 8 等）に準拠しているか
