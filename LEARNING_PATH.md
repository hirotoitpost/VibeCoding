# VideCoding ラーニングパス

## 目的
このドキュメントは、**Vibe Coding の全体像を段階的に習得するための詳細なラーニングパス**を提供します。

---

## フェーズ2: 理論学習 - 詳細ガイド

### 2.1 アイスブレーク：Vibe Coding とは？

#### 学習目標
- Vibe Coding の定義を理解
- 従来開発との比較ポイントを把握
- AI活用のメリット・デメリットを認識

#### 学習内容
1. **定義**
   - 自然言語指示と AI 実装の組み合わせ
   - 高速プロトタイピングのパラダイム
   - 人間とAIの役割分担モデル

2. **従来開発との比較**
   | 項目 | 従来開発 | Vibe Coding |
   |------|--------|------------|
   | 指示言語 | プログラミング言語 | 自然言語 |
   | 実装速度 | 数日〜数週間 | 数時間〜数日 |
   | 前提知識 | 高い | 低い |
   | 品質保証 | 高度 | 中程度（要検証） |
   | 複雑性 | 高度な案件に適す | 中程度まで |

3. **AI ツール分類**
   - コード生成系: GitHub Copilot, Claude Code
   - IDE 統合系: Cursor
   - 汎用: ChatGPT + 手作業

#### 学習時間
**推定 2-4 時間** (リサーチ + 実験)

#### 成果物
- `docs/vibe_coding_theory.md` - 定義と特性まとめ

---

### 2.2 ツール調査とハンズオン

#### 学習目標
- 各ツールの特性を実体験で把握
- ツール選定の判断基準を習得
- 効果的な指示文を書く練習

#### ツール別学習

##### 2.2.1 GitHub Copilot
**特徴**:
- IDE 内での自動補完
- コンテキスト理解が優れている
- 無料版あり（機能限定）

**学習タスク**:
1. インストール・設定
2. 基本的なコード補完を体験
3. 複数言語でのテスト

**推定時間**: 1-2 時間

---

##### 2.2.2 Cursor
**特徴**:
- Vibe Coding 専用 IDE
- 自然言語でのリファクタリング対応
- チャットインターフェース統合

**学習タスク**:
1. インストール
2. チャット型指示の実験
3. 複数ファイルの同時編集

**推定時間**: 2-3 時間

---

##### 2.2.3 Claude Code / Claude API
**特徴**:
- 推論能力が高い
- 複雑な要件理解に強い
- API 経由で自由な統合が可能

**学習タスク**:
1. API キーの取得
2. 基本的なプロンプト設計
3. 複数ラウンドの対話実験

**推定時間**: 2-3 時間

---

```
入力: 都市名（日本語）

指示例: 「OpenWeatherMap のAPIを使って、
東京の3日間の天気予報を日本語で取得する
Pythonスクリプトを作って。エラーハンドリングも付けて」
```

### 3.2 初級プロジェクト（難易度: ⭐⭐）

#### プロジェクト例群

**オプション A: 簡易 Web アプリ**
```
「シンプルな家計簿 Web アプリ」
- フロントエンド: React
- バックエンド: Node.js/Express
- データベース: SQLite

課題: 複数サービスの連携が必要
```

**オプション B: IoT スタブ**
```
「温度・湿度ロガー（シミュレーター）」
- ターゲット: Raspberry Pi/Arduino（シミュレーション）
- 言語: MicroPython / Arduino C
- 課題: ハードウェア理解、リソース制限
```

**オプション C: その他**
```
- CLI ツール化
- チャットボット
- データ解析スクリプト
```

#### 技術スタック候補
ユーザーの関心分野に応じて選択：
- Web: React, Vue, Next.js
- IoT: Arduino, Raspberry Pi
- スクリプト: Python, Go

#### 学習ポイント
- 外部ライブラリの管理（npm, pip）
- データベース基礎
- エラーハンドリング（本番レベル）

#### **ここから見えてくる AI の限界**
- ❌ 初期環境構築（Node.js, Python インストール）
- ❌ 複雑なデバッグ
- ⚠️ パッケージ最新版との互換性問題
- ✅ ビジネスロジック実装
- ✅ UI 実装

---

### 3.3 中級プロジェクト（難易度: ⭐⭐⭐）

#### 関心分野別プロジェクト例

| 分野 | プロジェクト例 | 技術スタック | 課題の深さ |
|------|--------------|-----------|---------|
| **IoT** | スマート家電コントローラー | Raspberry Pi, Python, MQTT | デバイス間通信、リアルタイム処理 |
| **Web** | マイクロサービス構成 | Docker, Kubernetes, Node.js | インフラ理解の必要性 |
| **モバイル** | ネイティブアプリ | React Native / Flutter | デバイス固有機能の理解 |
| **スマートコントラクト** | シンプルなDApp | Solidity, Web3.js | ブロックチェーン理解 |
| **ドローン制御** |自動飛行コントローラ | DroneKit, Python | 物理系の知識 |
| **ネットワーク** | パケット分析ツール | Python, Scapy | ネットワークプロトコル理解 |
| **動画編集** | 自動字幕生成ツール | FFmpeg, AI モデル | 大規模な依存・環境構築 |

#### **ここから見えてくる AI の限界（深化）**
- ❌ インフラストラクチャ設計
- ❌ セキュリティ監査
- ❌ パフォーマンスチューニング（推定値）
- ⚠️ ハードウェア統合（デバイス仕様の理解が必要）
- ✅ アルゴリズム実装
- ✅ API 設計
